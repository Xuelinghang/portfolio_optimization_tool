In the first sprint, we managed to lay down the groundwork for the project by creating the GitLab repository and seeding it with all the files required. We had a clear project organization with frontend and backend directories and created some initial UML diagrams to describe our system structure. Besides, we developed paper prototypes and UI wireframes for the principal screens, which have already allowed us to build a tangible vision of the user interface. We also set up a straightforward testing framework and wrote a preliminary API integration plan, including potential data sources such as Yahoo Finance, Alpha Vantage, and CoinGecko.

From now on, the plan for next week is to progress towards developing a functional authentication system by making a simple login page and a registration page. This will allow us to diverge from prototyping and proceed with developing secure user authentication and data storage. We will start outlining backend endpoints that support these authentication capabilities as well. Reflecting on this sprint, we are grateful for the solid foundation that we established, despite running into issues with some UI design aspects and API integration needs. These lessons teach us the importance of early communication and iterative feedback, setting the stage for a better development process in the future sprint.
