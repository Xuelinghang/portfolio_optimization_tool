During the fifth sprint, our team made steady progress on backend development and successfully linked the portfolio data entry module with simulated market data. We built and tested the core functionality in the market_fetcher.py module, which retrieves mock asset prices—laying the groundwork for real-time API integration with services like Yahoo Finance or Alpha Vantage in the upcoming sprint.

The database schema was finalized this week, now supporting multiple asset types including stocks, crypto, bonds, funds, and ETFs. Users can enter assets manually or upload them via CSV, and the data is correctly stored in the SQLite database. We also refined the model definitions in models.py and implemented secure, token-based user authentication in auth.py, ensuring proper handling of registration, login, and session management.

On the frontend, we improved the design and functionality of the portfolio entry form. The dataEntry.js script now successfully packages user input as JSON and sends it to the backend. Joint testing showed that the submitted data is correctly validated, processed, and stored. The form now provides a smoother user experience with improved input validation, dynamic updates, and consistent styling across the interface.

In the next sprint, we plan to focus on developing core financial computations, including Sharpe ratio, expected return, and volatility. We also aim to begin implementing chart visualizations using Chart.js to display each portfolio’s risk-return profile. Additionally, we will begin conducting full integration testing to ensure consistency and stability across the entire user workflow—from registration to data analysis and visualization.
